# 总结｜从 0 到真正上线，我学到的几件事

这次从 0 搭建并上线 Chirpy + GitHub Pages，我最大的收获并不是掌握了某个技术，而是形成了一套 **更理性的起步方式**。

## 一、先让系统跑起来，比理解所有原理重要得多

与其一开始就追求“完全理解 Jekyll、Docker 或部署细节”，不如先把本地环境跑通、文章显示出来。只要系统能真实运行，后续的问题都会变成“有上下文的具体问题”，而不是抽象焦虑。

## 二、技术选择要服务于当前阶段，而不是未来的想象

Docker、主题深度定制、复杂的 CI 配置，并不是无用，但它们并不适合刚起步时引入。一开始选择更直观、更可控的方式，反而更有助于建立工程直觉。

## 三、Jekyll 的规则严格，但并不复杂

文章不显示，几乎总是命名、目录或时间格式的问题。这些规则一旦摸清，就会变成稳定且可预期的约束，而不是阻碍。

## 四、真正的成就感来自“系统落地”，而不是“教程看懂”

当我在浏览器里看到线上地址与本地完全一致的博客页面时，我意识到：完成一次从 0 到 1 的落地，比学会多少零散知识都更重要。这种能力会迁移到今后所有长期项目中。

## 最后：给自己设定一个“停止折腾”的约束

我刻意给自己加了一个约束：**上线之后，不再继续折腾技术，而是专注持续写作**。

因为博客的价值，不在于它“多漂亮、多先进”，而在于它是否真的承载了长期思考，并被不断使用。
